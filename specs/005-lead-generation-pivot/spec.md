# PRD: 리드 생성 수익 모델 전환

**Feature Branch**: `005-lead-generation-pivot`  
**Created**: 2025-12-25  
**Status**: Draft  
**Priority**: P0 (Critical Path)

## Executive Summary

평수 계산기의 수익 모델을 **AdSense 중심 → 리드 생성 중심**으로 전환한다.

| 지표 | 현재 (AdSense) | 목표 (Lead Gen) | 변화 |
|------|---------------|-----------------|------|
| 필요 트래픽 | 50,000 세션/월 | 5,000 세션/월 | **-90%** |
| 예상 수익 | ₩300,000/월 | ₩500,000~1,500,000/월 | **+67~400%** |
| 손익분기점 | 12개월 | 3개월 | **-75%** |
| 핵심 KPI | 세션 수, RPM | 전환율, 리드 단가 | - |

### 왜 지금인가?

1. **인프라 준비 완료**: `BudgetEstimator.tsx`, `useLeadForm.ts`, GA 이벤트 추적 이미 구현됨
2. **블로커 최소화**: Tally 폼 ID만 연결하면 즉시 리드 수집 시작 가능
3. **시장 타이밍**: 부동산 시장 불확실성으로 "전문가 상담" 수요 증가

## Problem Statement

### 현재 문제점

1. **AdSense 수익 구조의 한계**
   - CPC ₩200 × CTR 1% × 1,000 방문 = ₩2,000/일
   - 월 50,000 세션 달성까지 12개월+ 소요
   - Mediavine 50K 세션 요건 미충족 시 저수익 AdSense에 갇힘

2. **기술 우선 함정 (Technical Trap)**
   - 코드베이스는 정교하나 수익 파이프라인 미완성
   - Tally 폼 ID가 플레이스홀더 상태 → 모든 CTA가 "준비 중" 반환
   - 트래픽 0인 상태에서 기술 최적화에 과투자

3. **가치 사슬 비효율**
   ```
   현재: 방문자 → 계산 → 이탈 (가치 ₩200)
   목표: 방문자 → 계산 → 불안 자극 → 상담 신청 (가치 ₩10,000~50,000)
   ```

### 핵심 가설

> **"평수 계산기 사용자는 잠재적 부동산 거래자다. 계산 결과에서 파생되는 불안(비용, 대출, 인테리어)을 상담 수요로 전환할 수 있다."**

## User Personas & Journeys

### Persona 1: 신혼부부 예비 매수자 (Primary)

**Profile:**
- 나이: 28-35세
- 상황: 전세 만기 6개월 내, 매매 vs 재계약 고민
- 불안 요소: "34평이면 아이 낳아도 괜찮을까?", "대출은 얼마나 받을 수 있지?"
- 디바이스: 모바일 90%

**Current Journey:**
```
네이버 검색 "34평 몇 제곱미터" 
→ 계산기 사용 
→ 결과 확인 
→ 이탈 (가치 수집 실패)
```

**Target Journey:**
```
네이버 검색 "34평 몇 제곱미터"
→ 계산기 사용
→ 인사이트: "34평은 국민평형, 3인 가족에 적합"
→ BudgetEstimator: "84㎡ 기본 인테리어 5,100만원~"
→ 불안 자극: "대출 한도 확인해보시겠어요?"
→ Tally 폼: 연락처 입력
→ 리드 확보 (가치 ₩10,000+)
```

### Persona 2: 인테리어 고려 중인 입주 예정자

**Profile:**
- 나이: 32-45세
- 상황: 신축 아파트 입주 3-6개월 전
- 불안 요소: "평당 얼마가 적정 가격이지?", "업체 비교는 어떻게?"
- 디바이스: 모바일 70%, PC 30%

**Target Journey:**
```
검색 "25평 인테리어 비용"
→ 계산기로 유입
→ BudgetEstimator: "59㎡ 가성비 리모델링 3,750만원~"
→ CTA: "검증된 업체 3곳 비교 견적 받기"
→ Tally 폼: 평수, 예산, 연락처
→ 인테리어 업체 리드 (가치 ₩30,000~100,000)
```

### Persona 3: 대출 한도 확인 중인 매수자

**Profile:**
- 나이: 30-40세
- 상황: 매물 확인 중, 자금 조달 계획 수립 단계
- 불안 요소: "LTV 70%면 얼마까지 가능하지?", "DSR은?"

**Target Journey:**
```
검색 "5억 아파트 대출 한도"
→ 계산기 유입 (programmatic SEO)
→ BudgetEstimator: "예상 대출 한도 3.5억, 필요 자기자본 1.5억"
→ CTA: "무료 대출 상담으로 정확한 한도 확인"
→ Tally 폼: 희망 매매가, 연소득, 연락처
→ 대출 리드 (가치 ₩10,000~50,000)
```

## Data Model

### Tally Hidden Fields Schema

```typescript
interface LeadFormData {
  // 레퍼럴 추적
  ref?: string;           // UTM 캠페인 or 블로그 출처
  source?: string;        // 유입 채널
  origin_page: string;    // 폼 호출 페이지 URL
  timestamp: string;      // ISO 8601

  // 계산 컨텍스트
  pyeong?: number;        // 사용자가 입력한 평수
  sqm?: number;           // 계산된 제곱미터
  insight_label?: string; // 예: "34평형 (84㎡)"

  // 인테리어 견적 관련
  basic_interior?: number;    // 기본 인테리어 비용 (만원)
  premium_interior?: number;  // 프리미엄 인테리어 비용 (만원)

  // 대출 관련
  house_price?: number;   // 희망 매매가 (만원)
  max_loan?: number;      // 예상 대출 한도 (만원)
  need_cash?: number;     // 필요 자기자본 (만원)
}
```

## Success Metrics

### Primary KPIs

| 지표 | 정의 | 목표 | 측정 방법 |
|------|------|------|----------|
| 전환율 (CVR) | 리드 폼 제출 / 총 방문자 | > 2% | GA4 funnel |
| 리드 품질 (SQL Rate) | 파트너 응답 리드 / 총 리드 | > 50% | CRM 추적 |
| 리드당 수익 (RPL) | 총 수익 / 총 리드 수 | > ₩10,000 | 파트너 정산 |
| 폼 완료율 | 제출 / 폼 시작 | > 30% | Tally analytics |

### Secondary KPIs

| 지표 | 정의 | 목표 |
|------|------|------|
| CTA 클릭률 | CTA 클릭 / 계산 완료 | > 5% |
| 평균 세션 시간 | 페이지 체류 시간 | > 2분 |
| 반송률 | 단일 페이지 이탈 | < 60% |
| 폼 시작률 | 폼 오픈 / CTA 클릭 | > 80% |

### Revenue Model

```
월간 수익 = 방문자 × CVR × SQL Rate × RPL

예시 1 (보수적):
  5,000 × 2% × 50% × ₩10,000 = ₩500,000/월

예시 2 (목표):
  10,000 × 3% × 60% × ₩15,000 = ₩2,700,000/월

예시 3 (낙관적):
  20,000 × 4% × 70% × ₩20,000 = ₩11,200,000/월
```
